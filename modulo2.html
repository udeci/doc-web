Módulo 2 · Semanas 3–4

⚙️ Migración y Refactorización Segura

Evolucioná el código a una versión moderna y robusta sin romper producción. La IA asiste en cada paso: desde el análisis de dependencias hasta la generación de tests de regresión.

4

Modernización de Stack (The Migration Path)

1h 30min ▼
🎯 Objetivo

Aprender a evolucionar un proyecto existente sin romper producción, usando IA para analizar dependencias, planificar migraciones y ejecutar refactorizaciones controladas.

⏱️ Bloque 1 · Auditoría técnica automática (20 min)

Cada estudiante carga su repositorio en el IDE con IA y ejecuta:

Prompt — Migration Assessment
Analizá este proyecto y generá un informe de modernización con:

1. Versión actual del stack
2. Dependencias obsoletas
3. Librerías con riesgo de seguridad
4. Patrones legacy detectados
5. Quick wins de refactor (bajo esfuerzo, alto impacto)
6. Riesgos de migración
7. Plan incremental en fases
Documento resultado: /docs/migration-assessment.md
  • Versión actual → versión objetivo
  • Breaking changes identificados
  • Dependencias críticas mapeadas
  • Deuda técnica priorizada
  • Estrategia de rollback definida
⏱️ Bloque 2 · Refactorización asistida (30 min)

Ejemplos por stack tecnológico:

Python
  • requirements.txt → pyproject.toml
  • Sin typing → type hints
  • Validaciones manuales → Pydantic
  • Errores genéricos → excepciones específicas
JavaScript / Node
  • CommonJS → ES Modules
  • JS → TypeScript
  • Callbacks → async/await
  • Validaciones → Zod
Java
  • Java 8 → 21
  • Spring Boot upgrade
  • DTOs mejorados
  • Dependencias antiguas
Prompt — Refactorización modular
Refactorizá este módulo para hacerlo moderno y mantenible.

Objetivos:
- Tipado fuerte (type hints / TypeScript)
- Validación robusta de inputs
- Manejo explícito de errores
- Separación de responsabilidades
- Mantener compatibilidad funcional exacta
⏱️ Bloque 3 · Plan de despliegue seguro (25 min)
  • Blue/Green deployment: dos entornos idénticos, switch instantáneo
  • Canary release: rollout gradual al 5% → 25% → 100%
  • Feature flags: activar funcionalidades sin nuevo deploy
  • Rollback: vuelta atrás en <60 segundos
Prompt — Estrategia de migración a producción
Diseñá una estrategia de migración a producción para este cambio minimizando downtime.

Incluí:
- Pasos ordenados con validaciones
- Criterios de go/no-go
- Procedimiento de rollback
- Métricas a monitorear en cada etapa
📄 Entregables
  • /docs/migration-assessment.md
  • /docs/migration-plan.md
  • /refactored-module/ con código modernizado

Testing Autónomo y Cobertura

1h 30min ▼
🎯 Objetivo

Usar IA para generar y mantener tests automáticos inteligentes que protejan la lógica de negocio durante y después de la migración.

⏱️ Bloque 1 · Mapa de cobertura actual (20 min)
Prompt — Gap analysis de testing
Analizá este proyecto e identificá:

1. Módulos sin cobertura de tests
2. Funciones críticas de negocio
3. Escenarios de borde no considerados
4. Riesgos si esas funciones fallan
5. Prioridad de testing (qué testear primero)
Documento: /docs/testing-gap-analysis.md
  • Porcentaje de cobertura estimado por módulo
  • Lista priorizada de tests faltantes
⏱️ Bloque 2 · Generación automática de tests (35 min)
Prompt — Unit tests
Generá tests unitarios completos para este archivo.

Incluir:
- Casos felices (happy path)
- Casos con inputs inválidos
- Edge cases y límites
- Mocks necesarios bien documentados

Framework: pytest / jest / junit (según el proyecto)
Prompt — Integration tests
Generá tests de integración para validar este flujo completo:

- Autenticación → endpoint → validación → DB
- Incluir setup y teardown
- Cubrir casos de error de red y timeout
Frameworks cubiertos
pytest Jest JUnit xUnit
⏱️ Bloque 3 · Tests de regresión (20 min)
Prompt — Regresión post-migración
Diseñá tests que aseguren que la nueva implementación mantiene exactamente la lógica anterior.

Compará el comportamiento antes y después del refactor.
Bonus — Datos sintéticos
Generá 100 casos edge para este modelo de datos,
cubriendo tipos inválidos, strings vacíos,
valores límite y datos en idiomas con caracteres especiales.
📄 Entregables
  • /tests/unit/
  • /tests/integration/
  • /tests/regression/
  • /docs/testing-strategy.md

Seguridad Proactiva y Análisis de Vulnerabilidades

1h 30min ▼
🎯 Objetivo

Convertir a la IA en un auditor permanente de seguridad del proyecto, integrando escaneos automáticos en el ciclo de desarrollo.

⏱️ Bloque 1 · Escaneo automático de código (25 min)
Herramientas de escaneo
Semgrep Snyk OWASP Dependency-Check GitHub Advanced Security

Vulnerabilidades a buscar: SQL injection, secrets hardcodeados, XSS, auth débil, SSRF, permisos excesivos.

Prompt — Auditoría completa de seguridad
Auditá este repositorio buscando vulnerabilidades:

1. Código inseguro (injections, XSS, CSRF)
2. Librerías con CVEs conocidos
3. Secretos o credenciales expuestos
4. Malas prácticas de autenticación
5. Nivel de severidad de cada hallazgo
6. Propuesta de remediación para cada uno
⏱️ Bloque 2 · Corrección asistida (30 min)

La IA propone fixes concretos y los explica:

Antes — SQL injection vulnerable
query = f"SELECT * FROM users WHERE id={user_id}"
Después — Parametrizado y seguro
cursor.execute(
    "SELECT * FROM users WHERE id = %s",
    (user_id,)
)
Prompt — Corrección con explicación
Corregí esta vulnerabilidad manteniendo la compatibilidad funcional.
Explicá:
- Por qué era riesgosa
- Qué tipo de ataque habilitaba
- Cómo el fix lo previene
⏱️ Bloque 3 · Automatización en CI/CD (20 min)
Prompt — Pipeline de seguridad
Generá un pipeline de GitHub Actions que ejecute:

- Tests unitarios e integración
- Scan de dependencias (Snyk / OWASP)
- Análisis estático de código (Semgrep)
- Bloqueo automático si hay vulnerabilidades críticas
- Reporte de resultados como PR comment
📄 Entregables
  • /docs/security-audit.md
  • /docs/remediation-plan.md
  • /.github/workflows/security.yml

⚙️ Resultado del Módulo 2

  • Proyecto modernizado con código limpio, tipado y dependencias actualizadas
  • Suite de tests generada por IA con cobertura de regresión controlada
  • Pipeline de CI con escaneo de seguridad continuo y automático
  • Capacidad de migrar cualquier sistema legacy con red de seguridad completa
